Grant types / Flows

OAuth2

Authorization Code

Classic confidential-client server flow with a client secret. No PKCE.

 OAuth2OIDC

Authorization Code + PKCE

Modern default. Public client, no secret, code_verifier / code_challenge proof-of-possession.

 OAuth2

Client Credentials

Machine-to-machine. No user involved — the client authenticates with its own secret.

 OAuth2

Device Code

For TVs, CLIs, devices without a browser. User authorizes on a second device by entering a code.

 OAuth2

Refresh Token

Trade a long-lived refresh token for a fresh access token without re-prompting the user.

 DEPRECATED

Implicit Flow

Historical flow that returned tokens in the URL fragment. Replaced by Auth Code + PKCE.
