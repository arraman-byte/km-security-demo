Implicit Flow

DEPRECATED — do not use in new applications.
The Implicit grant returns tokens directly in the browser URL fragment, which means:
  • Tokens leak into browser history, referer headers, server logs.
  • No client authentication and no code-exchange step, so token replay is harder to prevent.
  • Refresh tokens are not issued.
OAuth 2.1 and the Browser-Based Apps BCP recommend Authorization Code + PKCE instead. Shown here only so you can see the difference.

Built originally for SPAs that couldn't safely keep a secret and couldn't easily perform a back-channel token exchange. PKCE and modern browser CORS handling made it obsolete.

Sequence

  1. Browser redirected to /authorize?response_type=token id_token&client_id=...&redirect_uri=...&nonce=....
  2. User logs in.
  3. Authorization server redirects to redirect_uri#access_token=...&id_token=...&token_type=Bearer&expires_in=....
  4. JavaScript on the callback page reads window.location.hash.
