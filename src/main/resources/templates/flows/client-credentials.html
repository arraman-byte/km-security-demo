Client Credentials (M2M)

Machine-to-machine. No user, no browser, no consent screen. The client authenticates with its own credentials and gets an access token representing itself (the "service account"). Used for backend jobs, cron tasks, microservice-to-microservice calls.

Sequence

  1. Client POSTs to /token with grant_type=client_credentials, client_id, client_secret.
  2. Keycloak validates credentials and returns an access_token (no id_token, no refresh_token).
  3. Client uses the token to call protected APIs as itself (subject is the service account).
