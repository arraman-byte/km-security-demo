Authorization Code + PKCE

The modern recommended browser/mobile flow. The client (public, no secret) generates a random code_verifier, derives code_challenge = BASE64URL(SHA256(verifier)), sends the challenge with the authorize request, and later proves possession by sending the verifier when exchanging the code for tokens. This binds the authorization code to the client instance and defeats code-interception attacks.

Sequence

  1. Client generates code_verifier (random 43-128 chars) and computes code_challenge.
  2. Browser is redirected to /authorize with code_challenge + code_challenge_method=S256.
  3. User logs in; Keycloak redirects back with ?code=....
  4. Spring exchanges the code at /token, sending the original code_verifier.
  5. Keycloak hashes the verifier and compares to the stored challenge; on match, returns id_token + access_token (+ refresh_token).
