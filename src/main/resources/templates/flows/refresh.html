Refresh Token

Access tokens are short-lived (often 1–15 minutes). The refresh_token is a long-lived credential the client can exchange for a fresh access_token without re-prompting the user. Refresh tokens are not sent to resource servers — only to the authorization server's /token endpoint.

Sequence

  1. Client POSTs to /token with grant_type=refresh_token, refresh_token, client_id (+ secret if confidential).
  2. Authorization server validates the refresh token and issues a fresh access_token (and optionally a rotated refresh_token).
  3. Client replaces the old access_token (and the old refresh_token if rotated).
