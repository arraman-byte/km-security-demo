Device Authorization Grant (RFC 8628)

For devices without a browser or with limited input (smart TVs, CLIs, IoT). The device shows a short user_code and a URL; the user authorizes on a phone/laptop. Meanwhile the device polls the token endpoint until the user finishes.

Sequence

  1. Device POSTs to /device_authorization with client_id.
  2. Server returns device_code, user_code, verification_uri, interval, expires_in.
  3. User opens the URL on another device, enters the user_code, logs in, approves.
  4. Device polls /token with grant_type=urn:ietf:params:oauth:grant-type:device_code; gets authorization_pending until approved, then tokens.
